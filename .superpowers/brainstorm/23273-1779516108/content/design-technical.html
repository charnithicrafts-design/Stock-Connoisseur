Data Model & Implementation

The technical blueprint for the Local Minimalist.

SQLite Schema
CREATE TABLE stocks (
  symbol TEXT PRIMARY KEY,
  name TEXT,
  market TEXT, -- 'US' or 'IN'
  currency TEXT, -- 'USD' or 'INR'
  added_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE snapshots (
  symbol TEXT,
  timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
  price REAL,
  pe_ratio REAL,
  market_cap REAL,
  sma_200 REAL,
  rsi REAL,
  score REAL, -- 0-100 Intimator score
  FOREIGN KEY(symbol) REFERENCES stocks(symbol)
);

CREATE TABLE events (
  id INTEGER PRIMARY KEY,
  symbol TEXT,
  event_type TEXT, -- 'EARNINGS', 'ANNUAL_REPORT'
  event_date DATE,
  notified BOOLEAN DEFAULT 0,
  FOREIGN KEY(symbol) REFERENCES stocks(symbol)
);

Implementation Strategy

  • PWA: Use vite-plugin-pwa for service worker management and offline support.
  • SQLite: Use @sqlite.org/sqlite-wasm with the Worker API to keep the UI responsive during heavy calculations.
  • CORS Proxy: Use a lightweight public proxy (like cors-anywhere or a simple Cloudflare Worker) to fetch yfinance data.
  • Notifications: Use the Notification API triggered by a service worker alarm/timer when the app is backgrounded.
